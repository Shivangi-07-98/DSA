class Solution {
  public int lengthOfLIS(int[] nums) {
    int len = 1;

    int[] tails = new int[nums.length];
    tails[0] = nums[0];

    for (int i = 1; i < nums.length; i++) {
      if (nums[i] > tails[len - 1]) {
        len++;
        tails[len - 1] = nums[i];
      } else {
        int idx = Arrays.binarySearch(tails, 0, len - 1, nums[i]);

        if (idx < 0) {
          idx = -idx;
          idx = idx - 1;
        }

        tails[idx] = nums[i];
      }
    }

    return len;
  }
}

Explanation:

If the result is positive, the element is present in the array. The returned value represents its position (1-based indexing).

For example:

* 10 → 3 means 10 is present at the 3rd position. Since arrays use 0-based indexing, its array index is 2.
* 15 → 4 means 15 is present at the 4th position. Its array index is 3.

If the result is negative, the element is not present in the array. The absolute value of the result indicates the position where the element should be inserted to keep the array sorted.

For example:

* 12 → -4 means 12 is not present. It should be inserted at the 4th position. Its array index would be 3.
* 16 → -5 means 16 is not present. It should be inserted at the 5th position. Its array index would be 4.

Rule:

* Positive result → Element found at that position.
* Negative result → Element not found.
* Absolute value of the result → Position where the element exists or should be inserted.
* Position is 1-based, while array indexing is 0-based.
